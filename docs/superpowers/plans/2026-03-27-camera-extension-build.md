# Camera Extension Build & Bundle Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automate building the macOS camera extension and assembling the Lodestone `.app` bundle so `cargo build && ./scripts/bundle.sh` produces a launchable app with a working virtual camera.

**Architecture:** An Xcode project compiles the existing Swift files into a `.systemextension` bundle. A Cargo `build.rs` calls `xcodebuild` automatically. A shell script assembles the final `.app` with the Rust binary + extension embedded. The Rust app activates the extension at startup via `objc2` FFI.

**Tech Stack:** Xcode/xcodebuild, Cargo build scripts, codesign, objc2 (Rust FFI to SystemExtensions.framework)

**Spec:** `docs/superpowers/specs/2026-03-26-camera-extension-build-design.md`

```rust
use std::env;
use std::process::Command;

fn main() {
    // Only build the camera extension on macOS
    if env::var("CARGO_CFG_TARGET_OS").unwrap_or_default() != "macos" {
        return;
    }

    // Tell Cargo when to re-run this script
    println!("cargo:rerun-if-changed=lodestone-camera-extension/Sources/main.swift");
    println!("cargo:rerun-if-changed=lodestone-camera-extension/Sources/Provider.swift");
    println!("cargo:rerun-if-changed=lodestone-camera-extension/Sources/Device.swift");
    println!("cargo:rerun-if-changed=lodestone-camera-extension/Sources/Stream.swift");
    println!("cargo:rerun-if-changed=lodestone-camera-extension/Info.plist");
    println!("cargo:rerun-if-changed=lodestone-camera-extension/LodestoneCamera.entitlements");
    println!("cargo:rerun-if-changed=lodestone-camera-extension/LodestoneCamera.xcodeproj/project.pbxproj");

    // Check that xcode-select points to Xcode.app, not just CommandLineTools
    let xcode_path = Command::new("xcode-select")
        .arg("-p")
        .output()
        .expect("Failed to run xcode-select. Is Xcode installed?");

    let path = String::from_utf8_lossy(&xcode_path.stdout);
    if !path.contains("Xcode.app") {
        panic!(
            "xcodebuild requires Xcode, but xcode-select points to: {}\n\
             Run: sudo xcode-select -s /Applications/Xcode.app/Contents/Developer",
            path.trim()
        );
    }

    // Map Cargo profile to Xcode configuration
    let profile = env::var("PROFILE").unwrap_or_else(|_| "debug".to_string());
    let xcode_config = if profile == "release" { "Release" } else { "Debug" };

    let status = Command::new("xcodebuild")
        .arg("-project")
        .arg("lodestone-camera-extension/LodestoneCamera.xcodeproj")
        .arg("-scheme")
        .arg("LodestoneCamera")
        .arg("-configuration")
        .arg(xcode_config)
        .arg("-derivedDataPath")
        .arg("target/xcode-build")
        .arg("-quiet")
        .arg("DEVELOPMENT_TEAM=9YR7S3S6AS")
        .arg("build")
        .status()
        .expect("Failed to run xcodebuild. Is Xcode installed?");

    if !status.success() {
        panic!("xcodebuild failed with status: {}", status);
    }
}
```

```bash
#!/usr/bin/env bash
set -euo pipefail

# Usage: ./scripts/bundle.sh [--debug]
# Assembles Lodestone.app from the Cargo build output and Xcode-built camera extension.

BUILD_TYPE="release"
if [[ "${1:-}" == "--debug" ]]; then
    BUILD_TYPE="debug"
fi

XCODE_CONFIG="Release"
if [[ "$BUILD_TYPE" == "debug" ]]; then
    XCODE_CONFIG="Debug"
fi

REPO_ROOT="$(cd "$(dirname "$0")/.." && pwd)"
APP_BUNDLE="$REPO_ROOT/Lodestone.app"
RUST_BINARY="$REPO_ROOT/target/$BUILD_TYPE/lodestone"
EXTENSION_SRC="$REPO_ROOT/target/xcode-build/Build/Products/$XCODE_CONFIG/com.lodestone.camera-extension.systemextension"

# Verify inputs exist
if [[ ! -f "$RUST_BINARY" ]]; then
    echo "Error: Rust binary not found at $RUST_BINARY"
    echo "Run: cargo build$([ "$BUILD_TYPE" = "release" ] && echo " --release")"
    exit 1
fi

if [[ ! -d "$EXTENSION_SRC" ]]; then
    echo "Error: Camera extension not found at $EXTENSION_SRC"
    echo "Run: cargo build (build.rs should trigger xcodebuild)"
    exit 1
fi

echo "Assembling Lodestone.app ($BUILD_TYPE)..."

# Clean previous bundle
rm -rf "$APP_BUNDLE"

# Create bundle structure
mkdir -p "$APP_BUNDLE/Contents/MacOS"
mkdir -p "$APP_BUNDLE/Contents/Resources"
mkdir -p "$APP_BUNDLE/Contents/Library/SystemExtensions"

# Generate host Info.plist
cat > "$APP_BUNDLE/Contents/Info.plist" <<'PLIST'
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>CFBundleIdentifier</key>
    <string>com.lodestone.app</string>
    <key>CFBundleName</key>
    <string>Lodestone</string>
    <key>CFBundleDisplayName</key>
    <string>Lodestone</string>
    <key>CFBundleExecutable</key>
    <string>lodestone</string>
    <key>CFBundleVersion</key>
    <string>1</string>
    <key>CFBundleShortVersionString</key>
    <string>0.1.0</string>
    <key>CFBundlePackageType</key>
    <string>APPL</string>
    <key>LSMinimumSystemVersion</key>
    <string>13.0</string>
    <key>NSHighResolutionCapable</key>
    <true/>
    <key>NSCameraUsageDescription</key>
    <string>Lodestone needs camera access for video capture.</string>
    <key>NSMicrophoneUsageDescription</key>
    <string>Lodestone needs microphone access for audio capture.</string>
    <key>NSSystemExtensionUsageDescription</key>
    <string>Lodestone installs a virtual camera so other apps can use your composited output.</string>
</dict>
</plist>
PLIST

# Copy Rust binary
cp "$RUST_BINARY" "$APP_BUNDLE/Contents/MacOS/lodestone"

# Copy camera extension
cp -R "$EXTENSION_SRC" "$APP_BUNDLE/Contents/Library/SystemExtensions/"

# Sign inner bundle first (extension), then outer (app)
echo "Signing camera extension..."
codesign --force --sign "Apple Development" \
    --entitlements "$REPO_ROOT/lodestone-camera-extension/LodestoneCamera.entitlements" \
    --options runtime \
    "$APP_BUNDLE/Contents/Library/SystemExtensions/com.lodestone.camera-extension.systemextension"

echo "Signing app bundle..."
codesign --force --sign "Apple Development" \
    --entitlements "$REPO_ROOT/Lodestone.entitlements" \
    --options runtime \
    "$APP_BUNDLE"

# Verify
echo "Verifying signatures..."
codesign --verify --deep --strict "$APP_BUNDLE"

echo "Done: $APP_BUNDLE"
```

```rust
//! Activates the Lodestone camera extension via OSSystemExtensionManager.
//!
//! On first launch, macOS will prompt the user to approve the extension
//! in System Settings → Privacy & Security.

use objc2::rc::Retained;
use objc2::runtime::AnyObject;
use objc2::{define_class, extern_class, extern_protocol, msg_send, AllocAnyThread};
use objc2_foundation::{NSObject, NSObjectProtocol, NSString};
use std::ffi::c_void;

// --- Raw FFI for dispatch_get_main_queue (not in objc2) ---

extern "C" {
    fn dispatch_get_main_queue() -> *const c_void;
}

// --- FFI bindings for SystemExtensions.framework ---

extern_class!(
    #[unsafe(super(NSObject))]
    #[name = "OSSystemExtensionRequest"]
    pub struct OSSystemExtensionRequest;
);

unsafe impl Send for OSSystemExtensionRequest {}
unsafe impl Sync for OSSystemExtensionRequest {}

impl OSSystemExtensionRequest {
    pub fn activation_request(
        identifier: &NSString,
        queue: *const c_void,
    ) -> Retained<Self> {
        unsafe {
            msg_send![
                Self::class(),
                activationRequestForExtension: identifier,
                queue: queue
            ]
        }
    }

    pub fn set_delegate(&self, delegate: &AnyObject) {
        unsafe {
            let _: () = msg_send![self, setDelegate: delegate];
        }
    }
}

extern_class!(
    #[unsafe(super(NSObject))]
    #[name = "OSSystemExtensionManager"]
    pub struct OSSystemExtensionManager;
);

impl OSSystemExtensionManager {
    pub fn shared() -> Retained<Self> {
        unsafe { msg_send![Self::class(), sharedManager] }
    }

    pub fn submit_request(&self, request: &OSSystemExtensionRequest) {
        unsafe {
            let _: () = msg_send![self, submitRequest: request];
        }
    }
}

// --- Delegate implementation ---

define_class!(
    #[unsafe(super(NSObject))]
    #[name = "LodestoneExtensionDelegate"]
    struct ExtensionDelegate;

    unsafe impl ExtensionDelegate {
        #[unsafe(method(request:didFinishWithResult:))]
        fn _request_did_finish(&self, _request: &AnyObject, result: isize) {
            match result {
                0 => log::info!("Camera extension activated successfully"),
                1 => log::info!("Camera extension will complete after reboot"),
                _ => log::warn!("Camera extension finished with unknown result: {result}"),
            }
        }

        #[unsafe(method(request:didFailWithError:))]
        fn _request_did_fail(&self, _request: &AnyObject, error: &NSObject) {
            log::error!("Camera extension activation failed: {:?}", error);
        }

        #[unsafe(method(requestNeedsUserApproval:))]
        fn _request_needs_user_approval(&self, _request: &AnyObject) {
            log::info!(
                "Camera extension needs user approval. \
                 Open System Settings → Privacy & Security to approve."
            );
        }

        #[unsafe(method(request:actionForReplacingExtension:withExtension:))]
        fn _request_action_for_replacing(
            &self,
            _request: &AnyObject,
            _existing: &AnyObject,
            _replacement: &AnyObject,
        ) -> isize {
            1 // OSSystemExtensionReplacementActionReplace
        }
    }
);

impl ExtensionDelegate {
    fn new() -> Retained<Self> {
        let this = Self::alloc();
        unsafe { msg_send![super(this), init] }
    }
}

/// Request activation of the Lodestone camera extension.
///
/// This is idempotent — if the extension is already active, macOS treats it as a no-op.
/// On first run, macOS will prompt the user to approve in System Settings.
pub fn activate_camera_extension() {
    let identifier = NSString::from_str("com.lodestone.camera-extension");

    // Get the main dispatch queue (raw FFI, not in objc2)
    let queue = unsafe { dispatch_get_main_queue() };

    let request = OSSystemExtensionRequest::activation_request(&identifier, queue);

    let delegate = ExtensionDelegate::new();
    request.set_delegate(&delegate);

    // Keep delegate alive — leak it intentionally (it's a singleton for the app's lifetime)
    std::mem::forget(delegate);

    let manager = OSSystemExtensionManager::shared();
    manager.submit_request(&request);

    log::info!("Submitted camera extension activation request");
}
```

**Important — objc2 FFI risk and contingency:**

The `extern_class!` / `define_class!` macros from `objc2` 0.6 are the right tools for wrapping `SystemExtensions.framework`, but since these types aren't in any `objc2-*` crate, the exact macro syntax may need adjustment during implementation. The delegate uses `define_class!` with raw method implementations (matching the pattern in `src/gstreamer/screencapturekit.rs`) rather than `extern_protocol!` conformance, since the protocol isn't available from any crate.

**If the objc2 approach proves too difficult**, the fallback is:
1. Create a small `lodestone-camera-extension/activation.swift` file (~20 lines) that exposes a C-callable function
2. Compile it via `build.rs` using `swiftc` into a static library
3. Link it from Rust via `extern "C"`

This fallback is simpler but adds a second Swift compilation step to `build.rs`.
